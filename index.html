Einführung

Sass ist eine Erweiterung für CSS. Sie bietet viele Funktionen und hat eine sehr ähnliche Syntax wie CSS selbst. Es soll CSS nicht umändern, sondern lediglich das ausbessern, was bei CSS fehlt. Es ist eine grosse Hilfe für das Beibehalten der Übersichtlichkeit. Es werden auch Variablen mit Sass mitgebracht, welche das wiederholte Benutzen von Werten vereinfacht. Unsere Wissensquelle ist die Seite: Sass Guidelines

Bei Bedarf könnt ihr euch auch noch in Sass Guidelines reinlesen. Bei der Seite gibt es eine gute Kurzzusammenfassung von den wichtigsten Informationen über Sass.

CSS ist eine einfache Sprache. Da Sass beabsichtigt CSS zu schreiben, sollte es niemals komplexer als normales CSS werden. Das KISS Prinzip (Keep it simple, stupid) ist in diesem Fall der Schlüssel, und hat unter gewissen Umständen sogar vorrang über DRY (Don’t repeat yourself).

Syntax

Die Syntax von Sass hat das Grundprinzip von CSS übernommen, jedoch hat Sass gewisse Dinge vereinfacht. Eines dieser Dinge wäre, dass es keine Semikola und geschweiften Klammern mehr braucht.

  • Eine Einrückung wird mit zwei Spaces gemacht, nicht mit Tabs.
  • Der Code sollte so einfach wie möglich gehalten werden.

Kompilieren

Das Kompilieren ist nichts anderes als die Umwandlung von .sass oder .scss in CSS-Dateien. Sobald man das Sass geschrieben hat und man es als CSS-Datei haben will, kompiliert man die Datei.

Am besten geht die Kompilierung mit einer Erweiterung für Visual Studio Code, wie z.B
Live Sass Compiler
Diesen kann man ganz einfach auf der Internetseite herunterladen und in Microsoft Visual Studio Code installieren. Der Live Sass Compiler hat die Funktion, eine CSS-Datei zu generieren und diese direkt beim Abspeichern der Sass-Datei zu aktualisieren. Damit diese Funktion läuft, muss man bei Microsoft Visual Studio auf "Watch" klicken. Danach sollte die CSS-Datei erstellt werden und die Sass-Datei hat nun eine direkte Verbindung zur CSS-Datei.

Nesting

Nesting ist die Verschachtelung, wie wir bei HTML haben. Diese Funktion bringt Sass auch mit sich. Viele Entwickler, die grosse Seiten schreiben, benutzen ein Stylesheet wie Sass, um die Übersicht mit der Verschachtelung zu verbessern.

In diesem Beispiel sieht man, dass das a in Sass unter dem .class untergeordnet ist. Bei CSS ist es jedoch seperat aufgelistet. Dies hilft uns, wenn wir wissen, wo ein Element untergeordnet ist. Dann finden wir das gesuchte Element leichter.

Variablen

Variablen erlauben es uns, Werte ganz leicht wieder zu verwenden. Hierbei legt man einen Wert fest und einen Ansprechterm. Diesen Term kann man dann benutzen um diesen Wert anzusprechen.

Es sollten folgende Bedingungen zutreffen, damit die Erstellung einer Variablen Sinn macht.

  • Der Wert wiederholt sich mindestens zweimal
  • Der Wert wird vorraussichtlich einmal aktualisiert

Mixins

Mixins erlauben es dem Autor durch sein gesamtes Stylesheet hinweg Styles wiederzuverwenden, ohne auf nicht-semantische Klassen wie .float-left zurückzufallen.

Mixins können ganze CSS-Regeln beinhalten. Auch Parameter und Funktionen sind möglich. Viele Entwickler halten sich hier nicht an K.I.S.S (Keep it simple and stupid). Sollte ein Mixin länger als 20 Zeilen sein, sollte man es lieber in kleinere Teile aufteilen.

Grundlagen

Es gibt eine Faustregel, die man bei Mixins beachten sollte. Wenn Ihnen auffällt, dass eine bestimmte Gruppe von CSS Werten immer zusammen auftritt (also nicht zufällig), dann können Sie ein Mixin daraus machen.

Parameterlose Mixins

Manchmal werden Mixins nur dazu benutzt, um zu verhindern dass man eine Gruppe von Deklarationen immer und immer wieder wiederholt. Dennoch brauchen sie keine Parameter oder haben sensible Standardwerte welche nicht notwendigerweise einen Parameter benötigen. In solchen Fällen können wir beim Aufruf ohne Probleme die Klammern weglassen. Das @include Keyword (oder + Zeichen in der Intended-Syntax) verhält sich bereits als ein Indikator dafür dass die Zeile ein Mixinaufruf ist; es werden keine extra Klammern benötigt.

Inherit

@extend ist eine Funktion von Sass, mit der Klassen einen Satz von Eigenschaften miteinander teilen können. Bei Auswahlelementen, die eine Klasse in Sass erweitern, wird deren Auswahl direkt neben der Klasse eingefügt, die erweitert wird, was zu einer durch Kommas getrennten Liste führt.

Das sieht dann so aus: @extend .class-name;

Loops

Da Sass komplexe Datenstrukturen wie Listen und Maps anbietet, ist es keine Überraschung das es Autoren ebenfalls die Möglichkeit gibt über diese zu iterieren.

each

Der @each-Loop ist definitiv der meist genutzte aller von Sass zur Verfügung gestellten Loops. Es bietet eine saubere API um über eine Liste oder Map zu iterieren.

for

Der @for-Loop ist in Kombination mit der CSS Pseudoklasse :nth-* hilfreich. Außer in diesem Fall, solltest du eher einen @each-Loop verwenden wenn du über etwas iterieren musst.
